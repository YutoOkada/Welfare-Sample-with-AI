# 障害福祉の現場 AI活用事例ビューア

障害福祉の現場で「すぐに試せる」AI活用事例を検索・閲覧できる対話型Webアプリケーションです。管理者機能により、事例の追加・編集・削除が可能です。

## 🎯 プロジェクト概要

本システムは、障害福祉の現場スタッフが、自身の「職種」「事業所種別」「業務内容」に即したAI活用事例を即座に見つけられるように設計されています。

### 主な機能

#### ✅ 現在実装済みの機能

1. **事例の検索・閲覧機能**
   - フリーワード検索（タイトル、事例内容、実績、福祉的配慮を横断検索）
   - 職種、事業所種別、業務内容による絞り込み
   - 事例カードのクリックで詳細表示（モーダル）

2. **ダッシュボード機能**
   - 該当事例数の表示
   - 事業所種別ごとの事例分布を棒グラフで可視化（Chart.js使用）

3. **管理者機能（パスワード保護：0325）**
   - 事例の新規追加
   - 既存事例の編集
   - 既存事例の削除（確認ダイアログ付き）
   - 管理者モードの切り替え（パスワード認証）

4. **データ永続化**
   - RESTful Table APIによるデータベース連携
   - 追加・編集・削除したデータはリロード後も保持

### 技術スタック

- **フロントエンド**: HTML5, CSS3 (Tailwind CSS via CDN), Vanilla JavaScript
- **データ可視化**: Chart.js (事業所種別分布グラフ)
- **データストレージ**: RESTful Table API
- **デザイン**: レスポンシブデザイン（モバイル対応）

## 🚀 機能詳細

### 1. 事例閲覧機能

#### アクセスパス
- **メインページ**: `index.html`

#### 機能
- **検索バー**: 事例のタイトル、内容、実績、福祉的配慮を全文検索
- **フィルター**:
  - 職種（サービス管理責任者、支援員、相談支援専門員など）
  - 事業所種別（全般、就労移行支援、B型作業所、グループホームなど）
  - 業務内容（個別支援計画、日報・引継ぎ、記録、SSTなど）
- **リセットボタン**: 全てのフィルターと検索をクリア
- **カード表示**: 
  - タイトル、事例概要、職種、業務内容を表示
  - ホバー時にカードが浮き上がるアニメーション
  - クリックで詳細モーダルを表示

#### 詳細モーダル
- 具体的な活用事例
- 実績（業務改善）
- ただの効率化ではない部分（福祉的配慮）
- 動画集（YouTube URLなど）

### 2. 管理者機能

#### アクセス方法
1. 画面右上の「管理者モード」ボタンをクリック
2. パスワード入力モーダルが表示される
3. パスワード「**0325**」を入力
4. 管理者モードが有効化される

#### 管理者モード有効時の機能

##### 事例の追加
1. フィルターセクションに表示される「+ 新規追加」ボタンをクリック
2. 編集フォームが表示される
3. 必須項目を入力:
   - タイトル
   - 職種
   - 事業所種別
   - 業務内容
   - 具体的な活用事例
   - 実績（業務改善）
   - 福祉的配慮
   - 動画URL（任意）
4. 「保存」ボタンで追加完了

##### 事例の編集
1. 事例カードの「編集」ボタンをクリック
2. 編集フォームに既存データが読み込まれる
3. 内容を修正
4. 「保存」ボタンで更新完了

##### 事例の削除
1. 事例カードの「削除」ボタンをクリック
2. 確認ダイアログが表示される
3. 「OK」で削除完了（キャンセル可能）

### 3. データベース構造

#### テーブル: `cases`

| フィールド名 | 型 | 説明 |
|------------|-----|------|
| id | text | 事例ID（システム生成UUID） |
| title | text | 事例のタイトル |
| role | text | 職種 |
| service_type | text | 事業所種別 |
| task | text | 業務内容 |
| useCase | rich_text | 具体的な活用事例 |
| result | rich_text | 実績（業務改善） |
| welfare | rich_text | 福祉的配慮 |
| videoUrl | text | 動画URL（任意） |

#### システムフィールド（自動管理）
- `gs_project_id`: プロジェクト識別子
- `gs_table_name`: テーブル名
- `created_at`: 作成日時（ミリ秒）
- `updated_at`: 更新日時（ミリ秒）

## 🔌 API エンドポイント

### RESTful Table API

#### 事例一覧取得
```
GET tables/cases?page=1&limit=1000
```

#### 事例詳細取得
```
GET tables/cases/{record_id}
```

#### 事例作成
```
POST tables/cases
Content-Type: application/json

{
  "title": "事例タイトル",
  "role": "支援員",
  "service_type": "全般",
  "task": "日報・引継ぎ",
  "useCase": "具体的な活用事例...",
  "result": "実績（業務改善）...",
  "welfare": "福祉的配慮...",
  "videoUrl": "https://..."
}
```

#### 事例更新
```
PUT tables/cases/{record_id}
Content-Type: application/json

{
  "title": "更新後のタイトル",
  ...
}
```

#### 事例削除
```
DELETE tables/cases/{record_id}
```
